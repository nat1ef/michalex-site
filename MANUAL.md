# Οδηγός συντήρησης — site Μηχανουργείου Αλεξανδράκης

Πρακτικός οδηγός για το πώς αλλάζεις κείμενα, εικόνες και βασικά στοιχεία στο site,
χωρίς να χρειάζεται να ξέρεις προγραμματισμό. Κράτα το αρχείο αυτό στον φάκελο του
project — δεν εμφανίζεται πουθενά στο ίδιο το site, είναι μόνο για σένα.

---

## 1. Πώς βλέπεις τις αλλαγές σου ζωντανά, πριν τις δημοσιεύσεις

1. Άνοιξε το PowerShell και μπες στον φάκελο του project:
   ```
   cd C:\Users\manos\Desktop\site
   ```
2. Τρέξε:
   ```
   npm run dev
   ```
3. Περίμενε να δεις `Local: http://localhost:3000` και άνοιξέ το στον browser.
4. Άφησε το παράθυρο ανοιχτό. Κάθε φορά που αποθηκεύεις αλλαγή σε αρχείο μέσα στο
   `src\`, η σελίδα ανανεώνεται μόνη της.
5. Για να σταματήσεις: `Ctrl + C` μέσα στο terminal.

Το τρέξιμο τοπικά **δεν επηρεάζει** το πραγματικό site — βλέπεις τις αλλαγές μόνο εσύ,
στον υπολογιστή σου. Το site δημοσιεύεται μόνο όταν γίνει "push" (βλ. ενότητα 6).

## 2. Πού βρίσκεται κάθε κείμενο

**Το μεγαλύτερο μέρος του κειμένου είναι σε ένα αρχείο:**
`src\lib\content.ts` — άνοιξέ το με τον Notepad, το VS Code, ή όποιον editor έχεις.

| Ψάχνεις... | Ενότητα μέσα στο `content.ts` |
|---|---|
| Τηλέφωνο, Viber, διευθύνσεις, ωράριο, όνομα επιχείρησης | `siteConfig` (στην κορυφή του αρχείου) |
| Ετικέτες μενού (Αρχική, Υπηρεσίες...) | `pageNav` |
| Τα 4 μπλοκ υπηρεσιών (Γρανάζια/Τόρνος/Φρέζα/Ειδικές) | `serviceCategories` |
| Τα 3 βήματα «πώς δουλεύουμε» | `workSteps` |
| Λογότυπα πελατών («Μας εμπιστεύονται») | `trustedBy` |
| Οι 4 αριθμοί-στατιστικά (4.9, 21+, χρόνια, 100%) | `stats` |
| Κριτικές πελατών | `reviews` |
| Ετικέτες στο grid φωτογραφιών εργαστηρίου | `facilityBento` |
| Το έτος ίδρυσης (1983) | `FOUNDING_YEAR`, κοντά στην αρχή του αρχείου |

Ανοίγεις το αρχείο, αλλάζεις το κείμενο **μέσα στα εισαγωγικά** `"..."`, αποθηκεύεις.
Μην πειράξεις ό,τι είναι **έξω** από τα εισαγωγικά (κόμματα, άγκιστρα `{ }`, αγκύλες
`[ ]`) — αν κάτι σπάσει εκεί, η σελίδα θα βγάλει σφάλμα.

**Το υπόλοιπο κείμενο** (τίτλοι με ειδική μορφοποίηση, π.χ. μισή πρόταση σε μπλε
χρώμα) είναι γραμμένο απευθείας μέσα σε αρχεία components:

| Αρχείο | Τι περιέχει |
|---|---|
| `src\components\home\hero.tsx` | Τίτλος + υπότιτλος πρώτης οθόνης |
| `src\components\home\precision-showcase.tsx` | Κείμενο ενότητας «Ακρίβεια σε κίνηση» |
| `src\components\layout\site-footer.tsx` | Κείμενο υποσέλιδου |
| `src\components\layout\site-header.tsx` | Header / λογότυπο |
| `src\app\layout.tsx` | Τίτλος καρτέλας browser, περιγραφή SEO, keywords |
| `src\app\epikoinonia\page.tsx` | Σελίδα «Επικοινωνία» |

Πρακτικός κανόνας: αν είναι **δεδομένο** (τηλέφωνο, κριτική, αριθμός) → `content.ts`.
Αν είναι **τίτλος με μορφοποίηση** → μέσα στο αντίστοιχο component, ανάμεσα σε `<p>`,
`<h1>`, `<h2>` κ.λπ. tags — άλλαξε μόνο το κείμενο ανάμεσα στα tags, όχι τα ίδια τα tags.

### Κεφαλαία χωρίς τόνους

Σε πολλά σημεία (π.χ. λογότυπο, ετικέτες) το κείμενο γίνεται αυτόματα ΚΕΦΑΛΑΙΟ από
τον browser. Στα ελληνικά τα κεφαλαία **δεν παίρνουν τόνο**, οπότε το κείμενο-πηγή
είναι ήδη γραμμένο χωρίς τόνους (π.χ. `Μηχανουργειο` αντί για `Μηχανουργείο`). Αν
προσθέσεις καινούργιο κείμενο σε τέτοιο σημείο, γράψ' το κι εσύ χωρίς τόνους.

## 3. Πώς αλλάζεις ή προσθέτεις μια εικόνα

Όλες οι φωτογραφίες είναι μέσα στον φάκελο `public\images\`:

| Υποφάκελος | Τι περιέχει |
|---|---|
| `public\images\work\` | Πραγματικές φωτογραφίες δουλειάς — γρανάζια, άξονες, εργαστήριο |
| `public\images\backdrops\` | Φόντα (3D renders, θολές σκηνές εργαστηρίου) |
| `public\images\clients\` | Λογότυπα πελατών (SVG) |

**Για να αντικαταστήσεις μια υπάρχουσα φωτογραφία:** βάλε το νέο αρχείο στον ίδιο
φάκελο **με το ίδιο ακριβώς όνομα** (π.χ. αντικατέστησε το `gear-pile.jpg` με άλλη
φωτογραφία, ίδιο filename) — δεν χρειάζεται να αλλάξεις κώδικα.

**Για να προσθέσεις καινούργια φωτογραφία σε νέα θέση** (π.χ. νέα κάρτα στο grid του
εργαστηρίου): αντίγραψε το αρχείο μέσα στον σωστό φάκελο, μετά πήγαινε στο
`content.ts` και πρόσθεσε μια γραμμή με το νέο filename στο σωστό σημείο (π.χ. μέσα
στο `facilityBento` για το grid εργαστηρίου, ή στα `images` ενός `serviceCategories`
μπλοκ). Αντέγραψε τη μορφή μιας υπάρχουσας γραμμής για να δεις ακριβώς τι χρειάζεται.

**Συμβουλή μεγέθους:** οι φωτογραφίες εργασίας είναι γύρω στα 150-300KB η καθεμία.
Αν ανεβάσεις φωτογραφία απευθείας από κινητό (συνήθως 3-8MB), καλό είναι να τη
συμπιέσεις πρώτα (π.χ. squoosh.app) ώστε να μη φορτώνει αργά η σελίδα.

## 4. Πώς προσθέτεις ή αφαιρείς μια κριτική

Στο `content.ts`, βρες το `export const reviews = [ ... ]`. Κάθε κριτική είναι ένα
μπλοκ σαν αυτό:

```ts
{
  author: "Michail P.",
  rating: 5,
  text: "Άνθρωποι με γνώση και εμπειρία...",
  source: "Google",
},
```

- **Για να προσθέσεις:** αντέγραψε ένα ολόκληρο μπλοκ (από `{` μέχρι `},`) και
  επικόλλησέ το μέσα στη λίστα, άλλαξε `author`/`rating`/`text`.
- **Για να αφαιρέσεις:** διάγραψε ολόκληρο το μπλοκ, από το `{` μέχρι το `},`.
- Το `rating` είναι αριθμός από 1 έως 5 (αστέρια).

Οι κριτικές στο site κυλάνε μόνες τους αργά και σταματάνε όταν κάποιος βάλει το
ποντίκι πάνω τους — αυτό δουλεύει αυτόματα, δεν χρειάζεται να κάνεις τίποτα άλλο.

## 5. Τηλέφωνο, διεύθυνση, ωράριο, Viber

Όλα αυτά είναι στο `siteConfig`, στην αρχή του `content.ts`:

```ts
phone: "210 522 2541",
phoneHref: "tel:+302105222541",   // ίδιο νούμερο, μορφή για κλήση
viberNumber: "306907493500",
address: "Καστοριάς 2, Αθήνα 104 41",
hours: {
  weekdays: "Δευτέρα – Παρασκευή: 07:30 – 16:00",
  saturday: "Σάββατο: Κλειστά",
  sunday: "Κυριακή: Κλειστά",
},
```

**Πρόσεξε:** αν αλλάξεις το `phone`, άλλαξε και το `phoneHref` (ίδιο νούμερο, χωρίς
κενά, με `+30` μπροστά) — αλλιώς το κουμπί κλήσης θα καλεί λάθος νούμερο.

## 6. Πώς οι αλλαγές γίνονται live (δημοσίευση)

Το site ζει σε ένα σύστημα που λέγεται **git**, και δημοσιεύεται αυτόματα κάθε φορά
που στέλνουμε ("push") τις αλλαγές. Ο πιο απλός τρόπος: **ζήτα μου** (στο Claude) να
κάνω push μετά από κάθε αλλαγή που κάνεις — σου το επιβεβαιώνω και ενημερώνεται το
preview link σε 1-2 λεπτά.

Αν θες να το κάνεις μόνος σου, μέσα στο PowerShell στον φάκελο του project:

```
git add -A
git commit -m "Περιγραφή της αλλαγής"
git push origin redesign
```

Αν δεν είσαι σίγουρος/η τι έχει αλλάξει, τρέξε πρώτα `git status` για να δεις λίστα.

## 8. Πράγματα που να προσέχεις

- **Μη διαγράφεις** αρχεία μέσα στο `src\` αν δεν ξέρεις τι κάνουν — ρώτα πρώτα.
- Όταν αλλάζεις το `content.ts`, πρόσεχε τα εισαγωγικά `"..."` και τα κόμματα `,` στο
  τέλος κάθε γραμμής — αν λείπει κάτι, η σελίδα μπορεί να μη φορτώσει καθόλου.
- Μετά από κάθε αλλαγή, δοκίμασέ την πρώτα τοπικά (ενότητα 1) πριν ζητήσεις push.
- Αν κάτι "σπάσει" και δεν ξέρεις γιατί, πες μου τι άλλαξες και το φτιάχνουμε μαζί.
